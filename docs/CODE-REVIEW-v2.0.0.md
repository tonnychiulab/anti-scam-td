# 防詐迷宮 v2.0.0 程式碼審查報告

- 審查版本：v2.0.0
- 審查日期：2026-07-10
- 報告建檔：2026-07-12
- 審查方式：原始碼靜態審查、算術探針、本機瀏覽器重現
- 結論：不建議將受審的 v2.0.0 快照直接視為 release-ready

> 注意：本報告記錄的是 v2.0.0 審查當時的結果。工作區其後已由其他開發流程更新為 v2.0.1；README 雖宣稱已修補下列項目，但 v2.0.1 尚未經本次審查重新驗證，因此不得將本報告直接解讀為 v2.0.1 的現況。

## 一、審查摘要

v2.0.0 共確認 2 項高風險與 8 項中風險，另有數項低風險及發行品質缺口。其中「過關轉場重複結算」已在本機瀏覽器實際重現；Service Worker 的跨專案快取刪除則屬部署後可能影響同網域其他應用的高風險問題。

本次未找到可由玩家輸入觸發的 DOM XSS。JavaScript 語法、manifest JSON、四語資料結構、DOM id 參照、87 關路徑邊界與直橫轉置均通過檢查。

## 二、高風險問題

### H-01 過關轉場可重入，能重複領獎並跳關

**證據**

- `game.js:638-640` 在波次清空後將 `waveActive` 設為 `false`，再呼叫 `levelClear()`。
- `game.js:686-693` 每次呼叫都立即增加分數與點數，之後才啟動非同步轉場。
- `game.js:718-729` 等待 2.35 秒才執行下一關回呼。
- `game.js:1108-1110` 在轉場期間重新啟用「下一波」按鈕。
- `game.js:1357-1358` 只檢查 `!S.waveActive`，沒有檢查是否正在結算。
- `style.css:180-182` 的轉場層使用 `pointer-events:none`，底層控制仍可操作。

**實際重現**

第 1 關過關動畫中按一次「出怪」後：

- 分數由 105 增加為 160。
- 點數由 73 增加為 118。
- 關卡仍停留在第 1 關。

這證明同一關會再次執行結算。多次觸發還會排入多個 `afterQuiz()` 回呼，造成跳關、重疊測驗或狀態錯亂。

**建議**

加入明確的 `transitioning/phase` 狀態，使 `levelClear()` 冪等；下一波、畫布、商店、升級與支援技能都必須由狀態層統一閘控。UI 遮罩只能作為第二層防護。

### H-02 Service Worker 會刪除同 origin 其他應用的快取

**證據**

- `sw.js:14-18` 會刪除所有名稱不等於目前 `CACHE` 的 Cache Storage。
- Cache Storage 以 origin 隔離，而不是以 Service Worker scope 隔離。
- 正式部署路徑位於 `tonnychiulab.github.io/anti-scam-td/`，可能與同帳號其他 Pages 專案共用 origin。

**影響**

安裝或升級本遊戲時，可能清除同網域其他 PWA 的離線資源，造成跨專案資料破壞或離線失效。

**建議**

只刪除本應用自有前綴且不是目前版本的快取，例如限定 `asmd-` 前綴；加入建立無關 cache 後執行 activate 的遷移測試。

## 三、中風險問題

### M-01 三倍速／低幀率會漏判投射物與光束碰撞

- `game.js:546-548` 先限制原始幀差，再乘最高三倍速度，單幀遊戲時間可達 0.15 秒。
- `game.js:341-366` 的投射物只在移動前檢查距離，移動量不會 clamp 到目標。
- `game.js:558-570` 的光束只檢查移動後的位置，沒有跨幀掃掠判定。

算術探針顯示，`dt=0.15`、目標距離 60 時，投射物可在座標 0 與 78 之間永久振盪；光束也能在一幀內跨過完整碰撞帶而漏掉敵人。

建議保存前一位置並使用 swept collision；若投射物本幀移動距離足以抵達目標，應直接判定命中。

### M-02 減速與暈眩共用狀態欄位，效果會互相覆寫

- `game.js:356-357` 讓 slow 與 stun 共用 `slowLeft/slowPct`。
- `game.js:460` 的震撼彈也覆寫同一組欄位。
- `game.js:591` 只遞減單一計時器。

震撼彈的 2.5 秒完全暈眩可被後續雷達命中改成 45% 減速；相反順序則可能把 0.6 秒警察暈眩延長到既有減速時間。

建議分開保存 stun 與 slow 的時間及強度，移動速度計算時先處理 stun。

### M-03 手機轉向會遺失戰鬥狀態並解除手動暫停

- `game.js:1439` 直接清空投射物與已進入 50 秒冷卻的強光光束。
- `game.js:1443-1449` 在 600ms 後無條件把遊戲設為非暫停。
- `docs/DESIGN-v2.md:24-25` 宣稱轉向無損，並要求光束軸向互換。

玩家在手動暫停、光束施放中或短時間連續旋轉手機時，可能失去技能效果，或被舊計時器意外恢復遊戲。

建議保存轉向前暫停狀態、使用獨立 layout generation token，並轉置而不是刪除 beam／projectile。

### M-04 PWA 背景更新可能持續提供舊版資源

- `sw.js:31` 對所有請求使用 `ignoreSearch:true`，會把不同 query 版本視為同一資源。
- `sw.js:37-43` 的 `cache.put()` 沒有被 `await` 或 `return`。
- `sw.js:46` 的 `waitUntil(refresh)` 因此不保證快取寫入完成。

建議只在本版快取中精確比對 URL；若導航頁需忽略追蹤參數，應只對 navigation 明確正規化。快取寫入 Promise 必須納入 Service Worker 生命週期。

### M-05 核心遊戲無法使用鍵盤或螢幕閱讀器

- `index.html:96` 的 canvas 沒有 fallback、可存取名稱或焦點模型。
- `game.js:1236-1298` 的棋盤操作全部只綁定 pointer 事件。
- 各 overlay 沒有 dialog 語意、背景 `inert`、焦點限制或結果 live region。

鍵盤使用者可以選取武器按鈕，卻無法放塔、選塔或指定支援位置。建議提供可聚焦棋盤游標、方向鍵及 Enter／Space 操作，並加入等價的文字狀態介面。

### M-06 雙擊重設只判斷時間，不判斷位置或空白格

- `game.js:1201` 只保存 `lastTapAt`。
- `game.js:1285-1292` 僅以 350ms 與縮放倍率判斷雙擊。
- 第一擊會先執行 `handleTap()`，可能蓋塔、開選單或消耗支援技能。

建議保存前次點擊座標與目標，要求兩擊位於同一空白區且距離落在 tap-slop 內；或提供明確的重設視角按鈕。

### M-07 手機第一關提示描述不存在的操作流程

- `style.css:341-345` 在手機版隱藏所有武器按鈕，改成點草地開啟建造面板。
- `i18n.js` 四語的第一關提示仍要求先點武器、再點草地。

建議依 layout 顯示不同首關提示，並加入遊戲內可重新開啟的建造教學入口。

### M-08 最後一命結束後，同一幀仍能繼續得分

- `game.js:593-600` 最後一命耗盡後只跳出敵人迴圈。
- `game.js:602-627` 仍繼續處理塔、投射物與志工。
- `gameOver()` 已先建立畫面統計，但排行榜儲存時讀取的是之後的 `S.score`。

結束畫面分數可能和實際保存分數不同。建議最後一命觸發 `gameOver()` 後立即停止該幀更新，或於整幀結束後統一建立結算快照。

## 四、低風險與品質缺口

- `game.js:1555-1581`：「之後再說」與取消原生安裝都會永久停用 PWA 提示，文案與行為不符。
- `index.html:7`：`frame-ancestors` 放在 meta CSP 中不生效，無法提供所宣稱的防框架保護。
- `game.js:1297-1298`：`pointercancel` 與正常 `pointerup` 共用成功點擊流程，系統取消手勢也可能觸發操作。
- `game.js:1384`：排行榜數值消毒仍接受 `Infinity`。
- `game.js:543-545`：暫停與結束後仍持續每幀排程 RAF，行動裝置有不必要耗電。
- README 宣稱零資料收集／不呼叫 API，但頁面會連線 Google Fonts；應自架字型或修正文案。
- v2.0.0 發行內容沒有自動測試、CI 或可重現的人工驗收紀錄，與設計文件的「全量測試」描述不一致。
- 損命後完整重建波次，卻保留先前取得的點數、分數與擊殺數；若不是刻意的 comeback 設計，可利用剩餘生命重複刷獎勵。

## 五、通過項目

- `game.js`、`i18n.js`、`sw.js` JavaScript 語法檢查通過。
- `manifest.webmanifest` JSON 可解析，圖示尺寸符合 192×192、512×512、512×512 宣告。
- `APP_VERSION`、Service Worker cache、首頁及 README 的 v2.0.0 字串一致。
- 87 關路徑起訖、邊界、相鄰連續性與直橫轉置全部通過。
- 四語資料結構一致：13 塔、5 敵人、3 支援、12 題測驗、21 則提示。
- 75 個 HTML id 無重複，JavaScript 引用的 DOM id 均存在。
- 玩家名稱在寫入排行榜 DOM 前已有 HTML escape；其餘 HTML sink 來源為本地靜態字典或經消毒數值，未發現玩家可控 DOM XSS。

## 六、審查限制

- 受審目錄當時沒有 Git metadata，無法核對前代 diff、v2.0.0 tag 或線上部署 commit。
- 未在正式 HTTPS 站執行跨版本 Service Worker 升級、完整離線流程及 Lighthouse。
- 未進行實機 iPhone Safari、Android Chrome、safe area、pinch、旋轉及螢幕閱讀器驗收。
- 本報告不代表 v2.0.1 已通過複審；v2.0.1 應另跑完整回歸與針對上述項目的修補驗證。
