# v2.1.0 發佈驗證報告

- 版本：`v2.1.0`
- 分支：`agent/v2.1-mobile-ux`
- 基準：`15c477d`（v2.0.2 PC controls and regressions）
- 驗證日期：2026-07-13
- 規格來源：[UX-v2.1-mobile-guidance.md](./UX-v2.1-mobile-guidance.md)

## 結論

v2.1.0 已完成 D-01～D-10 的行動視覺導引與操作體驗實作。自動化回歸測試共 33 項全數通過；手機直向、手機橫向與桌機版面均無頁面溢位，瀏覽器主控台未出現錯誤或警告。桌機原有的分數、靜音與滑鼠右鍵防誤觸行為均保留。

本次驗證涵蓋桌面瀏覽器與手機尺寸模擬；正式上線前仍建議以至少一台實體 iPhone Safari 與一台 Android Chrome 做觸控、震動、瀏海安全區及效能抽查。

## 決策實作對照

| 決策 | 實作結果 |
| --- | --- |
| D-01 | v2.1 聚焦行動視覺導引與操作體驗，未擴張其他版本範圍。 |
| D-02 | 路線逐格揭示；第 1～3 關 1000ms、第 4 關起 700ms；減少動態時改為完整靜態顯示，玩家操作可提前完成揭示。 |
| D-03 | 不自動追蹤或縮放危險敵人；以敵人、出口與畫面邊緣聚合指示提醒，點擊邊緣指示只置中、不改變縮放。 |
| D-04 | 手機 HUD 保留暫停、速度與更多；靜音與資訊移入更多選單，開啟選單會暫停並在關閉後還原先前狀態。 |
| D-05 | 手機建塔採選塔、預覽射程與剩餘金額、確認建造的二段流程；餘額不足時不可確認。 |
| D-06 | 升級可直接執行並設 350ms 防連點；拆除採 3 秒內二次確認，顯示 70% 返還與 30% 損失。 |
| D-07 | 衝撞與電磁震撼採位置預覽與二次確認；施放前有 500ms 可取消短蓄力，實際施放後才進入冷卻。 |
| D-08 | 提供自動、完整、精簡三段品質；自動模式可降至內部最低安全層，依 FPS 門檻、波次與冷卻時間控制升降級。 |
| D-09 | 敵人路徑進度 80% 進入危險、92% 進入最終警戒、低於 75% 才解除；採每敵人與每波去重，無危險 2 秒後清除。 |
| D-10 | 手機常駐 HUD 不顯示總分，只顯示 800ms 合併得分提示；總分保留於更多資訊、過關、測驗、結算與排行榜。 |

## 自動化驗證

執行 `npm test`，結果為 **33/33 通過**。測試包含：

- 程式語法、版本號、快取版本與多語文字結構。
- 路線揭示時長、操作略過與減少動態行為。
- 危險門檻、遲滯解除、每敵人與每波去重。
- 支援技能預覽、取消及冷卻起算時機。
- 自動品質 FPS 門檻、升降級限制與最小間隔。
- 手機 HUD、更多選單暫停還原與分數隱藏。
- 建塔、升級、拆除的二段確認、防連點與暫停時即時重繪。
- 餘額更新、餘額不足停用與既有 v2.0.2 回歸項目。

手機直向、橫向與桌機測試頁的瀏覽器錯誤／警告紀錄皆為空。

## 測試中發現並修正

- 暫停時建塔、升級、拆除未立即更新畫面：改為操作成功後立即重繪。
- 戰鬥中塔選單餘額可能落後：選單顯示期間定期刷新可用金額。
- 餘額不足的建塔選項只有灰化、仍可操作：改為真正停用。
- 844 × 390 橫向畫面有 10px 垂直溢位：調整橫向畫布高度預算後消除。
