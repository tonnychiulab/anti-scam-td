# 多語 HTML 安全政策

本文件定義 `i18n.js` 與畫面輸出之間的信任邊界。目標是讓翻譯內容可以保留必要的排版標記，同時避免把一般字串誤當成 HTML。

## 預設規則

- 所有多語字串預設都是純文字，使用 `textContent`、表單屬性或等效的安全 DOM API 輸出。
- 只有下方白名單欄位可以送入 `innerHTML`。白名單內容屬於程式碼的一部分，必須與程式碼一起審查，不可由玩家、localStorage、網址、網路回應或第三方翻譯平台直接提供。
- 純文字若因既有版型必須進入 HTML 模板，必須先經 `escapeHtml()`；玩家名稱等外部或本機儲存資料也適用相同規則。
- 新增 HTML 欄位前，必須先更新本文件與自動測試，不能只在程式中新增例外。

## 允許 HTML 的欄位

| 欄位 | 用途 | 允許的標記 |
|---|---|---|
| `ui.tagline` | 首頁標語換行 | `<br>` |
| `ui.credit` | 公益聲明、強調與換行 | `<b>`、`<br>`、既有的 `<span class="en-sm">` |
| `ui.howList[]` | 玩法清單的重點強調 | `<b>` |
| `ui.loseComfort` | 結束畫面的安慰文字 | `<b>`、`<br>` |
| `ui.endWin` | 勝利結語 | `<br>` |
| `ui.endLose` | 失敗結語 | `<br>` |
| `ui.endStats` | 結算版型 | `<b>`、`<br>`，插入值仍須是數字或已轉義純文字 |

上述白名單只允許靜態、可閱讀的排版標記。即使位於白名單欄位，也禁止：

- `<script>`、`<iframe>`、`<object>`、`<embed>`、`<form>`、`<style>` 或外部資源標記。
- `javascript:`、`data:` 等可執行或內嵌內容網址。
- 任何 event handler 屬性，例如 `onclick`、`onerror`、`onload`。
- 任意 `style` 屬性、未列入白名單的 class、未知標籤或自訂屬性。

## 審查清單

1. 確認新字串是否真的需要 HTML；不需要時一律用 `textContent`。
2. 確認玩家輸入、localStorage、網址參數與網路資料不會直接流入 `innerHTML`。
3. 檢查四種語言的白名單欄位只有本文件允許的標記。
4. 執行 `npm test`，確認欄位結構、版本與 XSS 回歸檢查全部通過。
5. 若信任邊界有變更，在版本紀錄及發行驗證報告中明確記載。
