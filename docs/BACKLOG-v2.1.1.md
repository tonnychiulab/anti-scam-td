## P1｜優先修（信任與暫停語意問題）

### B-01　強光蓄力不得跨越暫停完成（原 M-210-03）
- **問題**：500ms 蓄力用原生 `setTimeout`，暫停或開啟「更多」選單期間仍會完成 → 消耗技能、起算冷卻。這是**暫停沒有真的凍結狀態**。
- **方向**：蓄力改由遊戲時間累積，或進暫停時保存剩餘時間、恢復後續算；選單關閉前不得偷偷施放。
- **驗收**：
  - 蓄力中暫停並超過 500ms，`useSupport` **不被呼叫**、冷卻**不起算**。
  - 恢復後可繼續蓄力或取消。
  - 新增一項「蓄力途中暫停」整合測試並通過。

### B-02　減少動態／最低品質時不得震動（原 M-210-04）
- **問題**：危險警告直接 `navigator.vibrate(40)`，未檢查 `reduceMotion` 或 `actualQuality==='minimum'`。使用者明確設定被忽略。
- **方向**：集中一個 haptic 出口,在減少動態、最低品質、或裝置不支援時一律跳過。
- **驗收**：
  - 開啟減少動態 → 危險警示**不觸發震動**。
  - 品質降到最低 → **不觸發震動**。
  - 新增回歸測試覆蓋兩種情境。

### B-03　音效偏好需持久化（原 M-210-05 部分）
- **問題**：`muted` 固定初始化為 `false`,切換靜音只改記憶體,重載／重開 PWA 後恢復有聲。
- **方向**：把音效納入 `asmd_ux_v21` 偏好並安全遷移舊資料。
- **驗收**：
  - 設為靜音 → 重載後仍為靜音。
  - 舊有偏好資料不會因新欄位而讀取失敗（有 `try/catch` 與預設值）。

## P2｜近期修（跨狀態流程與既有玩家更新）

### B-04　PWA 新版本交接（原 M-210-07）
- **問題**：SW 對同源 GET 一律 stale-while-revalidate,發新版後既有玩家首次進站可能整場停留舊版。**唯一會真實影響既有玩家的問題。**
- **方向**：導覽（HTML）請求改 network-first；或新 worker 接管後顯示「有新版本，點此更新」並於安全時機重載。
- **驗收**：
  - 模擬「舊 SW 控制中 → 部署新版」,玩家在**一次可見的提示或一次重載內**取得新版。
  - 新增「舊→新 worker 交接」整合測試（不只驗快取名稱）。

### B-05　手機手勢與建塔預覽互斥（原 M-210-01、M-210-02、D-05/D-07）
- **問題**：進入雙指手勢不會關建塔預覽；縮放後單指拖曳不平移；進入支援瞄準會直接清掉建塔預覽且無法還原。
- **方向**：建塔預覽與支援瞄準狀態分開保存；雙指手勢開始時關預覽、不扣點；縮放後單指超過 tap-slop 轉為 pan；技能取消時還原原建塔狀態。
- **驗收**：
  - 建塔預覽中雙指縮放 → 預覽關閉、點數不變。
  - 縮放後單指拖曳 → 視圖平移、不送出 tap。
  - 進入瞄準再取消 → 回到原本選定的塔種與格子。
  - 對應整合測試通過。

### B-06　建塔面板無障礙與失敗回饋（原 M-210-06）
- **問題**：面板開啟不移入焦點、完成／取消不回原格；塔種按鈕無 `aria-pressed`；取消鍵「✗」無 `aria-label`；二次驗證失敗直接關面板無提示。
- **方向**：保存來源格 → 開啟時聚焦面板 → 同步 `aria-pressed` → 取消鍵加多語 `aria-label` → 失敗時保留面板並以 live region 說明 → 完成／取消回到來源格。
- **驗收**：
  - 鍵盤開啟建塔面板後焦點在面板內；Esc／完成後回到來源棋盤格。
  - 螢幕閱讀器能讀出目前選中的塔與取消鍵名稱。
  - 驗證失敗時面板不消失且有語音提示。

## P3｜可排入但不阻斷（效能與資訊完整度）

### B-07　HUD 改為事件／dirty-flag 更新（原 L-210-08）
- **問題**：每個 animation frame 重查 13 個塔按鈕＋3 個支援按鈕並重寫 DOM,低階手機在大量敵人時增加耗電與掉幀。
- **方向**：改事件驅動；冷卻秒數等定時欄位降到 4–10Hz；快取固定 DOM 節點。
- **驗收**：靜態時 HUD 不再每幀寫 DOM；低階裝置實測掉幀改善（有前後數據）。

### B-08　升級面板補足資訊（原 L-210-09）
- **方向**：顯示下一級傷害／射程／攻速差異與扣款後餘額；MAX 維持不可操作。
- **驗收**：面板顯示「下一級數值 + 升級後餘額」；MAX 等級按鈕停用。

### B-09　路線／危險導引多語與宣告（原 L-210-10）
- **問題**：`#dangerEdge` 的 `aria-label` 寫死中文；路線揭示無 live-region 宣告。
- **方向**：新增四語字串；路線完成做一次 polite announcement；危險按鈕 accessible name 隨語系與數量更新。
- **驗收**：切英文後危險按鈕以英文朗讀；路線顯示後有一次「入口／出口／路線已顯示」宣告。

## 流程強化（跨版本，建議一併採用）

### B-10　發布打 git tag
- **問題**：目前無任何 git tag,只靠 commit message 標版,回溯「線上是哪一版」得比對 hash。
- **動作**：每次發布打 annotated tag（如 `v2.1.0`）。
- **驗收**：`git tag` 能列出各發布版；tag 指向實際部署 commit。

### B-11　CI 加入「LICENSE 必須存在」檢查
- **動作**：在既有 GitHub Actions 加一步驟,缺 `LICENSE` 即 fail。
- **驗收**：故意刪除 `LICENSE` 時 CI 轉紅。

### B-12　上線前真機與螢幕閱讀器抽查
- **問題**：33 項測試皆單元／靜態,跨狀態與觸控行為模擬器測不出。
- **動作**：至少一台實體 iPhone Safari ＋ 一台 Android Chrome 抽查觸控／震動／瀏海安全區／效能；一輪 VoiceOver 或 TalkBack。
- **驗收**：留下抽查紀錄（裝置、版本、結果）附於發布驗證文件。

## 建議實作順序

1. **B-00**（還原 LICENSE，發布前立刻做）
2. **B-01 → B-02 → B-03**（暫停語意與偏好遵守，信任優先）
