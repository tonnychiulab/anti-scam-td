# 工作日誌：2026-07-15

## 今日目標

依照 v2.1.0 程式碼審查建議，製作 v2.1.1 修補版，處理手機手勢、暫停蓄力、無障礙偏好、建塔流程及 PWA 更新問題。

## 完成內容

### 1. 手機手勢與建塔狀態

- 放大棋盤後，單指拖曳可平移畫面，不會被誤判為點擊。
- 雙指縮放或平移開始時會取消待處理點擊，並安全關閉建塔預覽。
- 支援技能暫時接管操作時會保存建塔狀態；取消支援瞄準或蓄力後，回復原本的塔種、位置與射程預覽。
- 支援技能成功施放後才會清除原本的建塔狀態。

### 2. 暫停與強光蓄力

- 強光手電筒的 500ms 蓄力由瀏覽器計時器改為遊戲時間計算。
- 遊戲暫停、開啟「更多」或其他會暫停遊戲的狀態下，蓄力會凍結。
- 恢復遊戲後才繼續剩餘蓄力，避免背景施放與冷卻誤啟動。

### 3. 偏好設定與無障礙

- 靜音狀態加入本機偏好保存，重新載入後仍會保留。
- 相容既有未包含靜音欄位的設定資料。
- 啟用「減少動態」或實際畫質降為最低時，不再觸發震動。
- 建塔面板加入塔種選取狀態、焦點回復與鍵盤焦點管理。
- 補上餘額不足、格子占用、塔未解鎖及目前不可建造等可朗讀錯誤訊息。
- 中文、英文、印尼文、越南文均補齊取消、錯誤與更新介面文字。

### 4. 建塔與更多選單

- 建造失敗時保留面板與預覽，不再直接關閉，方便玩家改選其他塔。
- 建造成功或取消後，焦點回到對應棋盤格。
- 手機「更多」選單新增「玩法說明」入口。

### 5. PWA 更新

- Service Worker 快取版本更新為 `v2.1.1`。
- HTML、JavaScript 與 CSS 等核心資源改為 network-first，離線時才回退快取，降低新舊程式混用風險。
- 新 Service Worker 等待或接管時會顯示「已有新版本可用」與「立即更新」。
- 加入 `SKIP_WAITING` 訊息處理及接管後安全重新載入流程。
- 遊戲、Service Worker、四語版本、HTML 與 `package.json` 已統一為 `2.1.1`。

### 6. PC 滑鼠選塔焦點框

- 同事回報 PC 網頁版點選下方塔種後，棋盤左上角會持續出現白色虛線框。
- 原因是商店按鈕把焦點交回棋盤後，自訂格線只檢查棋盤是否聚焦，沒有區分滑鼠與鍵盤焦點。
- 格線現在遵守瀏覽器的 `:focus-visible`：滑鼠選塔時不顯示，使用 Tab／方向鍵時仍保留無障礙提示。
- 新增滑鼠、鍵盤與棋盤失焦三種狀態的回歸檢查。
